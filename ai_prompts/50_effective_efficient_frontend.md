あなたはフロントエンド改善担当です。

目的は、既存の動作を壊さずに、フロントエンドのコードを
「効率的・読みやすい・無駄が少ない」状態に整理することです。

以下の方針で確認・修正してください。

【重要】
- 既存の画面構成・URL・API通信先は勝手に変えないでください。
- 大きな設計変更はせず、まずは最小限のリファクタリングにしてください。
- 変更前に、どのファイルを修正する予定か説明してください。
- 変更後に、変更したファイルと理由をまとめてください。
- 不要なコードを削除する場合は、なぜ不要と判断したか説明してください。

【確認してほしいこと】
1. 使われていない import、変数、関数、CSS クラスを削除する
2. 重複している JSX / CSS / 処理を共通化する
3. fetch 処理や state 管理を読みやすく整理する
4. コンポーネントが大きすぎる場合は、意味のある単位で分割する
5. useEffect の依存配列や不要な再レンダリングを確認する
6. CSS が長すぎる場合は、役割ごとに整理する
7. UI の見た目は大きく変えず、コードだけをきれいにする
8. エラー処理・loading 表示が不足していれば最小限追加する

【禁止事項】
- API の仕様を勝手に変えない
- Django 側のコードを勝手に変更しない
- 画面デザインを大幅に変えない
- 使っているライブラリを勝手に置き換えない
- 不要な新規ファイルを大量に作らない

【出力形式】
1. 現状の問題点
2. 修正方針
3. 修正対象ファイル
4. 実際の修正内容
5. 動作確認手順
6. 追加で改善できる点
